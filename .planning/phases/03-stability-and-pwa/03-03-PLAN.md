**Step A — Generate placeholder icon PNGs:**

The manifest and layout reference three PNG files that do not yet exist. Create them as simple dark-navy squares with "TE" text using a Node.js Canvas script. Run this script from the project root:

```javascript
// Run as: node generate-icons.mjs
import { createCanvas } from "canvas"
import { writeFileSync, mkdirSync } from "fs"

const sizes = [
  { size: 192, file: "public/icon-192x192.png" },
  { size: 512, file: "public/icon-512x512.png" },
  { size: 180, file: "public/apple-icon-180x180.png" },
]

for (const { size, file } of sizes) {
  const canvas = createCanvas(size, size)
  const ctx = canvas.getContext("2d")

  ctx.fillStyle = "#060b14"
  ctx.fillRect(0, 0, size, size)

  ctx.fillStyle = "#38bdf8"
  ctx.font = `bold ${Math.floor(size * 0.35)}px monospace`
  ctx.textAlign = "center"
  ctx.textBaseline = "middle"
  ctx.fillText("TE", size / 2, size / 2)

  writeFileSync(file, canvas.toBuffer("image/png"))
  console.log(`Created ${file}`)
}
```

If `canvas` is NOT available, use a simpler approach — create minimal valid PNG files using a pure Node.js approach with a 1x1 dark pixel PNG base64 pattern, scaled with sharp (if available), or create the smallest valid PNG binary manually.

If sharp is available:
```javascript
// Run as: node generate-icons.mjs
import sharp from "sharp"

const sizes = [192, 512, 180]
const files = {
  192: "public/icon-192x192.png",
  512: "public/icon-512x512.png",
  180: "public/apple-icon-180x180.png",
}

for (const size of sizes) {
  await sharp({
    create: {
      width: size,
      height: size,
      channels: 3,
      background: { r: 6, g: 11, b: 20 },
    },
  })
    .png()
    .toFile(files[size])
  console.log(`Created ${files[size]}`)
}
```

If NEITHER canvas nor sharp is available, create the PNG files as minimal valid PNGs using the smallest possible valid PNG binary. A 1x1 dark pixel PNG is sufficient as a placeholder — the file just needs to be a valid PNG so the browser doesn't 404. Use this approach:
```bash
# 1x1 dark PNG encoded as base64 — valid for all three sizes (placeholder only, final art in Phase 5)
python3 -c "
import base64, struct, zlib

def make_png(r, g, b, w=1, h=1):
    raw = b'\\x00' + bytes([r, g, b]) * w
    filtered = raw * h
    compressed = zlib.compress(filtered, 9)
    chunks = []

    def chunk(name, data):
        c = name + data
        return struct.pack('>I', len(data)) + c + struct.pack('>I', zlib.crc32(c) & 0xffffffff)

    chunks.append(chunk(b'IHDR', struct.pack('>IIBBBBB', w, h, 8, 2, 0, 0, 0)))
    chunks.append(chunk(b'IDAT', compressed))
    chunks.append(chunk(b'IEND', b''))
    return b'\\x89PNG\\r\\n\\x1a\\n' + b''.join(chunks)

data = make_png(6, 11, 20)
import os
os.makedirs('public', exist_ok=True)
for f in ['public/icon-192x192.png', 'public/icon-512x512.png', 'public/apple-icon-180x180.png']:
    open(f, 'wb').write(data)
    print(f'Created {f}')
" 2>/dev/null || echo "Python3 fallback failed"
```

Use whichever method succeeds. The files are placeholders — file size and visual quality do not matter for Phase 3. Final art will be swapped in Phase 5.
