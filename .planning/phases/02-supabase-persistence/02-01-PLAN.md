```sql
-- Phase 2: Supabase Persistence
-- Tables: trades, coaching_entries
-- RLS: enabled with users_own_* policies (USING + WITH CHECK)
-- Reference: .planning/phases/02-supabase-persistence/02-RESEARCH.md (Code Examples → Full RLS Migration SQL)

-- ============================================================
-- trades table
-- ============================================================
CREATE TABLE IF NOT EXISTS public.trades (
  id          TEXT PRIMARY KEY,
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  date        TEXT NOT NULL,
  time        TEXT NOT NULL,
  instrument  TEXT NOT NULL,
  direction   TEXT NOT NULL CHECK (direction IN ('long', 'short')),
  session     TEXT NOT NULL,
  contracts   INTEGER NOT NULL DEFAULT 1,
  pnl         NUMERIC NOT NULL DEFAULT 0,
  rmult       NUMERIC NOT NULL DEFAULT 0,
  outcome     TEXT NOT NULL DEFAULT 'breakeven' CHECK (outcome IN ('win', 'loss', 'breakeven')),
  confluences TEXT[] NOT NULL DEFAULT '{}',
  notes       TEXT NOT NULL DEFAULT '',
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

ALTER TABLE public.trades ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users_own_trades"
  ON public.trades
  FOR ALL
  TO authenticated
  USING ((SELECT auth.uid()) = user_id)
  WITH CHECK ((SELECT auth.uid()) = user_id);

CREATE INDEX IF NOT EXISTS trades_user_id_idx ON public.trades USING btree (user_id);

-- ============================================================
-- coaching_entries table
-- ============================================================
CREATE TABLE IF NOT EXISTS public.coaching_entries (
  id              TEXT PRIMARY KEY,
  user_id         UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  timestamp       TEXT NOT NULL,
  trade_count     INTEGER NOT NULL DEFAULT 0,
  title           TEXT NOT NULL DEFAULT '',
  full_content    TEXT NOT NULL DEFAULT '',
  archived        BOOLEAN NOT NULL DEFAULT FALSE,
  mode            TEXT NOT NULL DEFAULT 'analyze',
  market_snapshot TEXT NOT NULL DEFAULT '',
  patterns        TEXT NOT NULL DEFAULT '',
  process         TEXT NOT NULL DEFAULT '',
  risk            TEXT NOT NULL DEFAULT '',
  priority        TEXT NOT NULL DEFAULT '',
  momentum        TEXT NOT NULL DEFAULT '',
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

ALTER TABLE public.coaching_entries ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users_own_coaching"
  ON public.coaching_entries
  FOR ALL
  TO authenticated
  USING ((SELECT auth.uid()) = user_id)
  WITH CHECK ((SELECT auth.uid()) = user_id);

CREATE INDEX IF NOT EXISTS coaching_entries_user_id_idx ON public.coaching_entries USING btree (user_id);
```

Notes:
- `(SELECT auth.uid())` subquery form is the current Supabase recommendation (RESEARCH.md State of the Art) — evaluates once per query rather than per row.
- Both `USING` AND `WITH CHECK` are required (RESEARCH.md Pitfall 6) — `USING` covers SELECT/UPDATE/DELETE, `WITH CHECK` covers INSERT.
- `IF NOT EXISTS` makes the file idempotent — safe to re-run if needed.
- Column names use snake_case (Postgres convention); TypeScript camelCase mapping is handled in Plan 02.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser client → Supabase Postgres | Authenticated user JWT passed via PostgREST; RLS enforces user-scoping at DB level |
| Anonymous client → Supabase Postgres | Anon key with no JWT; RLS denies all rows because `authenticated` role is required |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `public.trades` SELECT | mitigate | RLS policy `USING ((SELECT auth.uid()) = user_id)` restricts SELECT to rows where the JWT-bound user_id matches; `TO authenticated` excludes anon role entirely |
| T-02-02 | Tampering | `public.trades` INSERT/UPDATE | mitigate | RLS `WITH CHECK ((SELECT auth.uid()) = user_id)` rejects any insert/update where the row's `user_id` does not match the authenticated user (Pitfall 6) |
| T-02-03 | Information Disclosure | `public.coaching_entries` SELECT | mitigate | Same RLS pattern as T-02-01 — `USING` clause + `TO authenticated` |
| T-02-04 | Tampering | `public.coaching_entries` INSERT/UPDATE | mitigate | Same RLS pattern as T-02-02 — `WITH CHECK` clause |
| T-02-05 | Elevation of Privilege | User row deletion cleanup | mitigate | `ON DELETE CASCADE` on `user_id` FK — when an auth.users row is deleted, all dependent trades/coaching rows are removed; prevents orphan-with-stale-user_id rows that could be re-claimed |
| T-02-06 | Tampering | `direction` / `outcome` enum corruption | mitigate | `CHECK` constraints reject any value outside the documented enum set |
| T-02-07 | Denial of Service | Unindexed user_id lookups under load | accept | Single-user app per D-09; `trades_user_id_idx` and `coaching_entries_user_id_idx` already cover the primary query pattern; further indexes deferred until Phase 4+ |
</threat_model>
