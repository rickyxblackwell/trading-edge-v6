# Phase 1 — UI Design Contract

> Visual and interaction contract for auth screens. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | base-nova |
| Component library | Radix (via shadcn) — override ALL default styling |
| Icon library | lucide-react |
| Font | Inter (UI/labels) · IBM Plex Mono (all numeric/data values) |

Source: `components.json` (detected). No third-party registries declared.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip padding, dot indicators |
| sm | 8px | Inline field padding, button icon gap |
| md | 16px | Card internal padding, field vertical padding |
| lg | 24px | Card padding (desktop), section gaps |
| xl | 32px | Page vertical centering offset |
| 2xl | 48px | Wordmark-to-card gap on auth pages |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Minimum tap target: 44px height on all buttons and interactive elements
- Input height: 48px (field height, not padding token — satisfies 44px touch target with room)
- Progress step dots: 8px diameter

---

## Typography

| Role | Size | Weight | Font | Line Height | Letter Spacing |
|------|------|--------|------|-------------|----------------|
| Body | 14px | 400 | Inter | 1.5 | normal |
| Label / Caption | 10px | 600 | Inter | 1.4 | 0.1em uppercase |
| Heading (card title) | 20px | 600 | Inter | 1.2 | -0.01em |
| Display (wordmark) | 13px | 600 | IBM Plex Mono | 1 | 0.2em uppercase |

Notes:
- Form inputs: 14px Inter weight 400, `color: var(--text)`
- Placeholder text: 14px Inter weight 400, `color: var(--text2)`
- Error messages: 12px IBM Plex Mono weight 400, `color: var(--red)`
- Helper/description text: 12px Inter weight 400, `color: var(--text2)`, line-height 1.5
- Input label above field: 10px Inter weight 600, uppercase, `color: var(--text2)`, letter-spacing 0.08em
- Settings section headers: 10px Inter weight 600, uppercase, `color: var(--text2)`, letter-spacing 0.1em

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` = #060b14 | Page background, behind orbs |
| Secondary (30%) | `var(--bg2)` = #0b1220, `var(--glass)` | Glass card surfaces, settings rows |
| Accent (10%) | `var(--accent)` = #38bdf8 | Primary CTA buttons, wordmark glow, focus rings, active step indicator, send button active state |
| Destructive | `var(--red)` = #f87171 | Logout button only |

Accent reserved for:
1. Primary "Sign in" and "Create account" CTA buttons
2. Wordmark glow (`box-shadow: 0 0 12px var(--accent), 0 0 24px rgba(56,189,248,0.3)`)
3. Input focus border (`border-color: rgba(56,189,248,0.45)`) and focus glow ring
4. Active step dot in 2-step signup progress indicator
5. Send button when input is populated
6. Step 2 "Gemini API Key" panel `border-accent` highlight
7. Settings API key status "Connected" badge

## Ambient Background — All Screens

All four screens (login, signup, coach preview modification, settings) share the exact same ambient orb background as the main app shell. The orbs are NOT re-implemented — reference the same `.ambient-bg` / `.orb` CSS classes from `globals.css`.

```
[position: fixed, inset: 0, z-index: 0, pointer-events: none]
  .orb.orb-1  — 700×700px accent blue, top-left, blur(100px), opacity 0.55
  .orb.orb-2  — 500×500px warm amber, bottom-right, blur(100px), opacity 0.55
  .orb.orb-3  — 420×420px purple, mid-center, blur(100px), opacity 0.55
```

Glass panels sit at `z-index: 10+` over this layer. The orbs must be visually perceptible through the glass.

### Step Transition Animation

When advancing from Step 1 to Step 2 (and back):
- Outgoing content: `opacity 1→0`, `translateX(0→-12px)`, duration `0.18s ease`
- Incoming content: `opacity 0→1`, `translateX(12px→0)`, duration `0.22s ease`
- Sequence: outgoing completes first, then incoming begins
- Implement with CSS classes + setTimeout or Framer Motion if available
- `@media (prefers-reduced-motion: reduce)`: instant swap, no transforms
